Projetos

Portal de indicadores Nitdata

Em dezembro de 2016 fui indicado pelo coordenador do Núcleo de Inovação Tecnlógica (NIT) UESC para desenvolver uma aplicação à pedido Rede NIT Nordeste. Esta aplicação tem como objetivo facilitar a consulta e divugação dos principais indicadores de desempenho dos núcleos de inovação tecnlógica da região Nordeste.

Um dos principais requisitos da aplicação é a apresentação visual dos resultados, de forma que coordenadores pudessem apresentar e discutir resultados fazendo uso direto da aplicação.

Ao analisar o escopo do projeto e o estado da arte relevante para a eventual aplicação, cheguei a conclusão que a aplicação poderia ser desenvolvida por apenas um desenvolvedor, desde que esteja munido das ferramentas adequadas. Assumi a responsabilidade como único desenvolvedor da aplicação.

Stack

  • Node.js e npm: Ambiente para programação do código-fonte
  • Vue.js: Framework para interfaces baseado na arquitetura React
  • Materialize: Componentes de interface baseados em Material Design.
  • SASS: Customização e manutenção do CSS
  • Chart.js: Componentes de gráficos estatísticos
  • Snap.svg: Manipulação de objetos SVG

Diferencial: Mapa interativo

O maior desafio deste projeto foi atender ao pedido de que a aplicação contesse um mapa interativo. O principal caso de uso desse pedido seria: ao se interagir com o mapa, deverão ser mostrados os indicadores da instituição selecionada. Durante o levantamento de tecnologias, nenhuma se mostrou adequada para a tarefa.

Analisando outras aplicações que faziam uso de mapas interativos, notei uma particularidade recorrente: o uso de imagens SVG. Passei a procurar então por tecnologias para trabalhar com tal formato. Acabei encontrado a biblioteca Snap.svg, que facilita muito a manipulação de elementos SVG atráves de javascript. Estudando as funcionalidades do Snap.svg me deu idéia então de como proceder.

Para atender este pedido, produzi um mapa da região Nordeste no Inkscape, de forma que cada instituição tivesse seu próprio elemento SVG. Foi então atribuído um identificador para cada elemento (estado ou instituição), para que pudesse ser manipulado através do javascript.

Com o mapa produzido, todos os outros elementos do projeto se encaixaram. Com os identificadores codificados na própria imagem foi possível detectar com qual elemento o usuário estaria interagindo, permitindo que a aplicação forneça as informações adequadas, assim como apresentar feedbacks visuais para a interação, como a mudança de cores e animações para hover e click.
